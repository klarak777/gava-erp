# Verziótörténet - V0.4.2

*Létrehozva: 2026-06-26*

## Összefoglalás

2026-06-26-tól a rendszer megváltozott elérési utakon hozza létre és keresi az új 25-26-os szezon dokumentumait.

---

## 1. Fuvarmegbízások – Új elérési út (25-26-os szezon)

**Régi mappa (24-25-ös és korábbi szezonok – változatlan):**
```
\\192.168.1.5\raktar\Fuvarok\Fuvarmegbízás\{szezonkód}\{fuvarozó_mappa}\{dok_neve}.docx
```

**Új mappa (25-26-os szezon, 2026-06-26-tól):**
```
\\192.168.1.5\raktar\MI Teszt\ERP Fuvarm\25-26\{fuvarozó_neve}\{dok_neve}.docx
```

**Példák:**
- `\\192.168.1.5\raktar\MI Teszt\ERP Fuvarm\25-26\BOGNÁR TRANS\BOGNÁR TRANS GHU 238.docx`
- `\\192.168.1.5\raktar\MI Teszt\ERP Fuvarm\25-26\KÓNYA TRANS\KÓNYA TRANS BEL 001.docx`

**Keresési logika:**
- Ha a DB-ben tárolt elérési úton (`file_path`) nem található a fájl, a rendszer fallback-ként dinamikusan megpróbálja az aktuális szezonhoz tartozó helyes elérési utat is.

---

## 2. EKAER dokumentumok – Új elérési út (25-26-os szezon)

**Régi mappa (24-25-ös és korábbi szezonok – változatlan):**
```
\\192.168.1.5\raktar\Fuvarok\EKAEREK\EKAEREK 2024-2025\{transport_number}\{rendszám}.docx
```

**Új mappa (25-26-os szezon, 2026-06-26-tól):**
```
\\192.168.1.5\raktar\MI Teszt\ERP EKAER\EKAEREK 2025-2026\{transport_number}\{rendszám}.docx
```

**Példa:**
- `\\192.168.1.5\raktar\MI Teszt\ERP EKAER\EKAEREK 2025-2026\BEL001\AI HK 730 - AA ID 874.docx`

> [!NOTE]
> A transport_number a kamionszámból képzett, szóköz nélküli formátum (pl. "BEL 001" → `BEL001`).

---

## 3. Automatikus mappa létrehozás – Új fuvarozók

Ha az Admin felületen egy **új fuvarozó céget** adnak hozzá, a szerver automatikusan létrehozza a 25-26-os ERP Fuvarm almappáját:
```
\\192.168.1.5\raktar\MI Teszt\ERP Fuvarm\25-26\{fuvarozó_neve}\
```

---

## 4. is_sent mező kezelése

- A Fuvarmegbízás generálásakor a `transport_orders` táblában az `is_sent` mező értéke automatikusan **`false`** (nem kiküldve).
- Az EKAER generálásakor az `ekaer_records` táblában az `is_sent` mező értéke szintén **`false`**.
- Bejelöléssel lehet `true`-ra változtatni, az adatbázis az állapotot megőrzi.
